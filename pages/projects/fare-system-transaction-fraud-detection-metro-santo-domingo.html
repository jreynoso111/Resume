Fare System Transaction Fraud Detection - Metro Santo Domingo

Category: Data Analytics / Fraud Detection / Transit Fare Systems

Project Overview

I conducted an internal analytical investigation to detect fraudulent activity within the Metro Santo Domingo fare collection system. The project objective was to identify irregular transaction behavior indicating that certain users had found a way to manipulate the payment medium. Using Power BI, I analyzed fare transaction records to detect abnormal card behavior. The investigation led to the identification of three users performing fraudulent activities that allowed them to bypass the normal transit card recharge process.

Context

  • The Metro operates a card-based fare payment model where users must recharge at authorized ticket offices or machines before travel.
  • Certain cards appeared with unexpected balance increases without a legitimate recharge transaction in the system.
  • This behavior indicated a potential vulnerability in the payment medium or in the fare workflow.

Analytical Approach

Power BI Transaction Modeling

Built a transaction model in Power BI to evaluate card lifecycle behavior and detect inconsistencies between expected and observed event sequences.

Behavioral Pattern Analysis

Analyzed relationships between Card ID, timestamp, station, transaction type, balance before and after transaction, recharge events, and fare validation events.

Critical Pattern Detection

Identified events where card balances increased without passing through authorized recharge points such as ticket offices or approved recharge terminals.

Investigation Logic

  • Cards receiving balance increases without recharge transactions.
  • Cards showing recharge amounts not associated with ticket office activity.
  • Transaction sequences inconsistent with normal passenger behavior.
  • Repeated anomalous patterns across multiple stations and time windows.

Outcome

The analysis successfully identified three fraudulent users exploiting a vulnerability in the payment system. Findings were documented and delivered to the appropriate internal teams to support corrective actions and security improvements within the fare system. This work demonstrated the importance of analytics in fraud detection and operational security for transit infrastructure.

Tools Used

  • Power BI
  • Data modeling
  • Transaction analysis
  • Pattern detection
  • Transit fare system analytics

Professional Impact

This project strengthened internal monitoring capabilities and highlighted how advanced analytics can protect critical transportation infrastructure from financial and operational vulnerabilities.
