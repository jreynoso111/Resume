Fleet Maintenance Analytics System

A full analytics ecosystem built to monitor, predict, and optimize maintenance operations for a commercial fleet. Includes automated data ingestion, cleaning pipelines, KPI dashboards, and anomaly tracking.

Project Overview

This system integrates multiple daily data sources — NetSuite exports, GPS activity logs, maintenance invoices, parts inventory, and internal tracking sheets — into a single analytical model that exposes operational patterns and financial impact.
It centralizes the full fleet lifecycle and highlights cost drivers, repeat repair patterns, and GPS inconsistencies, reducing manual work and giving leadership a clear, reliable base for decision‑making.

Main Features

  • Automated merging of 7+ daily NetSuite data extracts
  • Historical tracking for each VIN (location, repairs, balances)
  • Real‑time GPS status monitoring and movement analysis
  • Cost vs. downtime correlation
  • KPI engine for Pay KPI, Remaining Balance, Oldest Open Invoice
  • Inventory preparation pipeline tracking
  • Exception detection (stripped vehicles, repeated repairs, missing titles)

Key Components

Dashboard Suite

Power BI dashboards for maintenance cost analysis, part usage, downtime, GPS activity, and reliability KPIs.

Data Pipeline

Power Query transformations unify historical and current data sources, detect changes, and consolidate VIN‑level history.

Operational Tracking

Tracks inventory status, repairs, locations, GPS events, customer payments, and repo activity.

Operational Impact

This system reduced manual report preparation by over 90%, standardized daily fleet oversight, and enabled rapid detection of operational anomalies and financial risk. It also established a reliable backbone for future predictive maintenance modeling.
