Repo & Risk Monitoring System

Centralized repo and risk view for funded trucks and trailers, combining payment status, GPS behavior and repo activity to decide quickly which units to cure, recover or write off.

Project Overview

This system was built to remove the typical chaos around delinquent deals: spreadsheets that don't match, GPS portals disconnected from payment data, and repo updates scattered in emails. The result is a single view that tells you which units are at risk, which are already in the hands of a repo agent, and which recovered units are stuck in storage or pending sale.

Key Components

Risk classification

Flags every unit as on-time, at risk, in recovery, repossessed or sold, so there are no grey areas or forgotten deals.

Repo pipeline board

Tracks assignments to agents, pickup attempts, storage locations and sale status in a single pipeline instead of scattered updates.

GPS & behavior signals

Uses GPS events, movement patterns and offline periods as early-warning signals that complement pure payment data.

Main Features

  • Portfolio view of all delinquent and recovered units with clear risk stages.
  • Timeline of key events: missed payments, broken promises, GPS tampering, agent assignment, pickup and sale.
  • Basic filters by lender, agent, region, unit type and days late to focus on what matters first.
  • Simple status language so operations, finance and repo agents talk about the same thing.

Impact

With this repo and risk view, the team can see in minutes what used to take several spreadsheets, phone calls and separate portals. It reduces the chance of missing a risky unit, shortens the time between delinquency and recovery, and gives management a clear picture of where money is getting stuck: idle repos, long storage times or units recovered but not listed for sale.
