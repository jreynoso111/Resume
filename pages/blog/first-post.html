Why This Blog Lives in Code

For this site, each post is its own HTML page under pages/blog/. That keeps URLs stable, makes posts easy to version, and avoids runtime dependencies for the public experience.

The goal is simple: publish pages that load fast, look intentional, and remain easy to maintain.

Suggested Post Structure

A good post reads like an operations memo: short sections, clear headings, and a strong summary. Use visuals only when they actually add clarity.

Checklist

  • Title: specific and measurable.
  • Lead: 2–3 sentences that set context.
  • Body: sections with real examples.
  • Close: what to do next.

Publishing New Posts

To publish a new post, duplicate this file, rename it to your slug (for example pages/blog/my-post.html), then add a new card to pages/blog.html.
